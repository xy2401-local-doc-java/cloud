pcmOpSubscriptionTransitionPlan

Componentfm_subscription
Visibilitypublic
Transactionrequired

Description

This operation takes as input a source plan that specifies the plan currently owned by the account and a target plan that specifies the plan to transition to.

Array of results information This returns the event created for transition. Under PIN_FLD_RESULTS, There may be sub PIN_FLD_RESULTS arrays, Each sub PIN_FLD_RESULTS contains the output from the pcmOpSubscriptionCancelDeal first and then pcmOpSubscriptionPurchaseDeal return flists for each Service in the request. This will depend on the how the transition plan configured like how many Services in the Plan and how many Deals in each Service in each Plan. Example:- From Plan and To Plans contains 2 Services and each Service has one Deal. The Sample response will be as shown below. 0 PIN_FLD_POID POID [0] 0.0.0.1 /account 588793 13 0 PIN_FLD_RESULTS ARRAY [0] Results from Cancel Deal1(service 1) from Plan 0 PIN_FLD_RESULTS ARRAY [1] Results from Purchase Deal1a(service 1) from To Plan 0 PIN_FLD_RESULTS ARRAY [2] Results from Cancel Deal2(service 2) from Plan 0 PIN_FLD_RESULTS ARRAY [3] Results from Purchase Deal2a(service 2) from To Plan
