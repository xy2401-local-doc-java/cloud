pcmOpArBillDispute

Componentfm_ar
Visibilitypublic
Transactionrequired

Description

This operation dispute a bill amount by calling the item dispute operation. Following are the ways a bill can be disputed: The bill dispute amount can be specified. The amount can be a positive or a negative number. A negative amount credits a bill while a positive amount debits it. A bill can not be disputed beyond its total due. That is, the Due Amount should not change its sign. If we cannot actually dispute the entire dispute amount, the operation will fail. If the bill dispute amount is not mentioned then entire due amount is disputed. In case we want to specific amount should be allocated to which item, we need to specify the items with corresponding amounts. During the item transfer process, the status of the dispute item is changed to closed if it is open, which is the case normally. However, if the dispute item status is closed before item transfer, the item transfer will set the status to open. For items that has usage amount and tax, only the usage amount can be disputed. Disputing tax only Item is not allowed One of the following operation operation description messages can be returned to the caller: Cannot dispute the entire amount. Success. fm_ar_bill_dispute_validations: Check the input values. The input bill object must be AR bill object. Through with Validations. Bill Can be disputed. Tax only item cannot be dispute. fm_ar_bill_do_item_dispute: Error creating dispute item. The PIN_ERRCLASS_SYSTEM_DETERMINATE error buffer returned (if any) can be: PIN_ERR_BAD_OPCODE
