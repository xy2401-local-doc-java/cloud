0int    INCLUDE_CHILDRENIn Out
Userequired

Indicator as to whether child amounts to be included. Can have values of 0 - do not include child amounts or 1 - include child amounts or 2 - include only child amounts. If a hierarchy has multiple levels of children, and the above billinfo/bill is both a child billinfo and a parent billinfo, there are two cases: To get the details as a child billinfo as a part of the current hierarchy (in this case the value should be 0 ) or to get the details as a parent billinfo including its children ( in this case the value must be 1 )

0int    AMOUNT_INDICATORIn Out
Useoptional

The Amount indicator - 0 for credit, 1 for debit.The amount from and to does not have any sign

0decimal    AMOUNT_FROMIn Out
Useoptional

The Amount from range. If specified then used else not.This is used against PIN_FLD_ITEM_TOTAL

0decimal    AMOUNT_TOIn Out
Useoptional

The Amount to range. If specified then used else not. This is used against PIN_FLD_ITEM_TOTAL. The client tool shall validate that the range is valid if both are specified

0int    THRESHOLDIn Out
Useoptional

Threshold value for the number of action items to be retrieved. If the number fetched is less or equal to the thhresold then the response shall have the details of the items. If greater the response shall have a field PIN_FLD_THRESHOLD containing the actual number of items that acould be retrieved.

0enum    STATUSIn Out
Userequired

1 The Value 1 is for pending items i.e PIN_ITEMS_STATUS_PENDING
2 The Value 2 is for open items i.e PIN_ITEMS_STATUS_OPEN
4 The Value 4 is for closed items i.e PIN_ITEMS_STATUS_CLOSED
6 The Value 6 is union of PIN_ITEMS_STATUS_OPEN and PIN_ITEMS_STATUS_CLOSED.Gets the bill items for items which are either in PIN_ITEMS_STATUS_OPEN or PIN_ITEMS_STATUS_CLOSED
Status of the item.Values are PIN_ITEM_STATUS_PENDING (1), PIN_FLD_ITEMS_STATUS_OPEN (2), PIN_FLD_ITEM_STATUS_CLOSED (4), Union of PIN_ITEMS_STATUS_OPEN and PIN_FLD_ITEM_STATUS_CLOSED (6) gets the bill items for items which are either in PIN_ITEMS_STATUS_OPEN or PIN_ITEMS_STATUS_CLOSED. Note status 8(reversed) cannot be applied to bill_items

0timestamp    END_TIn Out
