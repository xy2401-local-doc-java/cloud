pcmOpArEventDispute

Componentfm_ar
Visibilitypublic
Transactionrequired

Description

This operation handles disputes according to given dispute nformation and events. It generates reservation notifications accordingly. Following are the ways that this operation works: If a back date is specified, it must be on or after the account and the general ledger (GL) posting date. The dispute process involves closing the bill with the event' bill-info, create and record events generated for the adjustment, transfers the fund to the original items, and perform reservation notification based on the tax flag. Whenever we need to call a resource reservation, a notification will be called accordingly to do the reservation. We invokes the pcmOpActUsage operation with the notify-only event using a PIN_OPFLG_CALC_ONLY flag. Event reservation objects are created for holding the dispute amount until their settlement. Notes on tax treatment: Notifications are called for all Non-Currency amounts. Notifications are called for Currency amounts only when the PIN_EVENT_ADJ_TAX_ONLY flag is not set. Notes on the operation flags: When the flags include the pcmOpflgCalcOnly flag , no fields in the database are changed and the event object is not actually created, but the fields that would have been used to create the event object are returned to the caller on the response. We explicitly specify the pcmOpflgCalcOnly flag to invoke the pcmOpActUsage operation. We explicitly specify the pcmTransOpenLockObj to open a transaction and lock the given account object. When the flags include the pcmOpflgReadResult flag, all the fields in the event object are returned on the response, not just the poid. One of the following operation operation description messages can be returned to the caller: Dispute not allowed since backdate is before account effective_t. Dispute not allowed since backdate is before GL posting date. RRF - an optional manager is not installed. Please refer to Oracle Monetization Cloud documentation. The PIN_ERRCLASS_SYSTEM_DETERMINATE error buffer returned (if any) can be: PIN_ERR_BAD_OPCODE
