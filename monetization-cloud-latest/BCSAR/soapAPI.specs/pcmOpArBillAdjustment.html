pcmOpArBillAdjustment

Componentfm_ar
Visibilitypublic
Transactionrequired

Description

This operation adjusts a bill by calling the item adjustment operation. Following are the different ways a bill can be adjusted: The bill adjustment amount may be specified. The amount can be a positive or a negative number. A negative amount credits a bill while a positive amount debits it. A bill can not be credited beyond its total due. However, there is no such limit on amount by which a bill can be debited. Optionally we may also specify the percentage of the bill due amount as the adjustment parameter. This parameter if specified overrides the adjustment amount. In case we want to specific amount should be allocated to which item, we need to specify the items with corresponding amounts. In case amount of any item returned by the bill-item search operation is 0, then the item will not be considered for adjustment. Notes on tax treatment: The operation reads the tax flag from optional field PIN_FLD_FLAGS specified in the request. If the value of the flag is PIN_AR_WITH_TAX then tax is calculated on the amount specified in the request. If the value of the field is PIN_AR_WITHOUT_TAX then only the amount specified in the request is adjusted. If the PIN_FLD_FLAGS optional field is not specified in the request then the system's default tax behavior (PIN_AR_WITH_TAX) is used. This setting is specified in the cm pin.conf as follows: - fm_ar tax_reversal_with_tax 1 #for without tax - fm_ar tax_reversal_with_tax 2 #for with tax By default this entry will not be there in cm's pin.conf and so the behaviour will be deferred taxation. One of the following operation operation description messages can be returned to the caller: The input bill object must be AR bill object. No adjustable items found. Tax only item cannot be adjusted. The PIN_ERRCLASS_SYSTEM_DETERMINATE error buffer returned (if any) can be: PIN_ERR_BAD_OPCODE

Poid of the Bill object to be adjusted. This must be an AR Bill object.

0decimal    AMOUNTIn Out
Userequired

The adjustment amount to be adjusted from the Bill. If the amount is greater than 0, it must be less than or equal to the total amount due of the Bill as the adjustment cannot be more than the amount due. The amount field can have a value of zero only if the percentage is specified or if the amount is specified within the items array.

0decimal    PERCENTIn Out
Useoptional

Adjustment percent of the Bill amount due. This is an optional parameter. If specified, it will override the value specified in the PIN_FLD_AMOUNT field. This is the percent of the net amount(i.e. amount without tax) of total amount due on the bill.
