pcmOpBillGroupMoveMember

Componentfinancial
Visibilitypublic
Transactionrequired

Description

This operation moves the specified account to (or out from) the group. In particular, it can move a member-account from one group to other.

0poid    POIDIn Out
Userequired
Class/account

Poid of account to be moved

0poid    PARENTIn Out
Useoptional
Class/account

Poid of account to be moved to (target). If it is not presented then the movement is provided out of the group.

0int    FLAGSIn Out
Useoptional

This flags is used to add validation which checks if any of the billinfo of input Account (to be moved) is member of Collections Group. Valid values are : PIN_COLL_GRP_VALIDATE (1) and PIN_COLL_DELETE_AND_MOVE (2) PIN_COLL_GRP_VALIDATE (1): If billinfo of input account (to be moved) is a member of collections group, return Collections group details. In this case, input account is not moved into Billing Hierarchy. PIN_COLL_DELETE_AND_MOVE (2): Delete member billinfo from Collections Group (if present) and continue with bill move member flow

0timestamp    WHEN_TIn Out
Useoptional

This field is used in order to provide a differed movement by creation of the schedule object (so, the real movement will not be done by the operation at this time). This field indicates the time when the deferred movement should be executed.

0string    DESCRIn Out
Useoptional
Max length255

This field should be used only if PIN_FLD_WHEN_T is specified. It may contain an arbitrary text which describes the deferred movement.
