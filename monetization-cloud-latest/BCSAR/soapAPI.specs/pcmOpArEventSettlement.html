pcmOpArEventSettlement

Componentfm_ar
Visibilitypublic
Transactionrequired

Description

This operation settles previous disputes according to given settlement information. It locates and releases previously reserved sources accordingly. Following are the ways that the operation works: If a back date is specified, it must be on or after the account and the general ledger (GL) posting date. The dispute process involves closing the bill with the event item and bill-info, create and record events generated for the settlement, transfers the settlement amounts to the original items, and perform reservation release notification based on the tax flag. Whenever we need to a release resource reservation, a notification will be called accordingly to release a previously reserved object. We invokes the pcmOpActUsage operation with the notification event using a PIN_OPFLG_CALC_ONLY flag. Event reservation objects that were created previously for holding the dispute amount will be released. Notes on tax treatment: Release Notifications are called for all Non-Currency amounts. Release Notifications are called for Currency amounts only when the PIN_EVENT_ADJ_TAX_ONLY flag is not set. Notes on the operation flags: We explicitly specify the pcmOpflgCalcOnly flag to invoke the pcmOpActUsage operation. That is , no fields in the database are changed and the event object is not actually created, but the fields that would have been used to create the event object are returned to the caller on the response. We explicitly specify the pcmTransOpenLockObj to open a transaction and lock the given account object. When the flags include the pcmOpflgReadResult flag, all the fields in the event object are returned on the response, not just the poid. One of the following operation operation description messages can be returned to the caller: Dispute not allowed since backdate is before account effective_t. Dispute not allowed since backdate is before GL posting date. The PIN_ERRCLASS_SYSTEM_DETERMINATE error buffer returned (if any) can be one of the following: PIN_ERR_BAD_OPCODE PIN_ERR_BAD_POID_TYPE PIN_ERR_BACKDATE_NOT_ALLOWED
