pcmOpArBillinfoWriteoff

Componentfm_ar
Visibilitypublic
Transactionrequired

Description

This operation writes off a bill-info by getting all the items for the bill-info object. Following are the ways a bill-info can be written off: The bill will be closed for the write-off. The bill items that are in open state will be searched and they should have non-zero due to write-off. Also the net due of all items must be greater than 0. The net and tax amounts corresponding to the items will be calculated to be written-off. Notes on the operation flags: When the flags include the pcmOpflgCalcOnly flag , the low-level operation will skip creating a write-off item, transferring the write-off amount, and writing the modified item object fields to the database. But the fields that would have been used to create the event object are returned to the caller on the response. When the flags include the pcmTransOpenLockObj flag, transactions will be opened to lock the bill-info and its item objects. When the flags include the pcmOpflgReadResult flag, all the fields in the event object are returned on the response, not just the poid. One of the following operation operation description messages can be returned to the caller: Write-off error : pending bills. Write-off error : unallocated items. The PIN_ERRCLASS_SYSTEM_DETERMINATE error buffer returned (if any) can be one of the following: PIN_ERR_BAD_OPCODE PIN_ERR_BAD_VALUE
