Migrating OTBI Enterprise From Test to Production

Transfer OTBI Enterprise system state and configurations from your stage environment to your production environment.

Use snapshots to migrate catalog objects. For more information about snapshots, see About Snapshots
To migrate your OTBI Enterprise service from test to production:
  1. Migrate the OTBI Enterprise service system state:
    1. In the test environment, log in to the Service Console and click New Snapshot in the Snapshots tab.
    2. In the New Snapshot dialog box, enter a Description for the system state. For example, you could enter the purpose of the snapshot, if it’s migrating from test to production.
    3. Click OK.
    4. Click the Manage Snapshot icon for the new snapshot and select Download.
    5. In the Snapshot Download dialog box, enter and confirm a Snapshot Password and click OK.
      The Snapshot is stored as a Business Intelligence Archive (BAR) file in the local area.
    6. Copy the BAR file from the stage environment to the production environment.
    7. In the production environment, log in to the Service Console and click Upload Snapshot.
    8. In the Upload Snapshot dialog box, click Browse.
    9. In the File Upload dialog box, locate the BAR file you copied from the stage environment and click Open.
    10. In the Upload Snapshot dialog box, enter the Snapshot Password and click OK.
      A confirmation stating “Snapshot Restore Completed Successfully” is displayed.

      Note:

      This replaces the custom catalog content, other than user folders, in the production environment.
  2. Migrate Configuration Manager setup data:
    1. In the test Environment, select Export Setup Data from the Tasks bar in Configuration Manager.
    2. Click Export Data to display the Export Data dialog box.
    3. In the Export Data dialog box, select all check boxes in the Setup Objects to Export list and click Export.
    4. Save the export file.
      The Setup Data file is stored in the local area.
    5. In the production environment, select Import Setup Data from the Tasks bar in Configuration Manager.
    6. In the Import Setup Data dialog box, specify the location and name of the Setup Data file you exported from the stage environment.
