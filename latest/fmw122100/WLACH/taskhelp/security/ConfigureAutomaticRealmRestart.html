Enable automatic realm restart


The ability to make non-dynamic configuration changes to a security realm and put them in effect immediately, without the need to restart WebLogic Server, is called automatic realm restart, or realm auto-restart.

However, note the following:
  • In the default security realm, automatic realm restart is disabled by default.
  • In new security realms that you create, automatic realm restart is enabled by default.

For more information, see Using Automatic Realm Restart.

To enable automatic realm restart for a security realm:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the left pane, select Security Realms.

    All the security realms available for the WebLogic domain are listed in the Realms table.

  3. Click the name of the security realm for which you want to enable automatic realm restart.
  4. In the Configuration > General page, select Automatically Restart After Non-Dynamic Changes.
  5. Click Save.
  6. In the Change Center, click Activate Changes.
