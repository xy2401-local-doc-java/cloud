Alterações na estrutura de menus

Todos os usuários antigos

As opções de navegação do serviço foram simplificadas. O que você vê depende da sua atribuição e de se o seu serviço é padrão ou empresarial.

  • As funcionalidades de gerenciamento de arquivos são mostradas clicando em Documentos.

  • As funcionalidades relacionadas a conversa são acessadas clicando em Conversas.

  • Todas as funcionalidades de construção de sites são acessadas clicando em Sites.

  • Os ativos digitais são mostrados clicando em Ativos.

  • Modelos, temas e layouts são acessados clicando em Desenvolvedor.

  • Para acessar estatísticas relacionadas a serviços, clique em Análise.

Se você tiver uma atribuição administrativa, as opções adicionais incluirão Definições, usada para definir configurações gerais de serviço, e Ativos, usada para configurar repositórios, tipos de conteúdo, canais e políticas de localização.

Pessoas, Grupos, Coleções, Itens de Conteúdo

Todos os usuários antigos

Estas funcionalidades agora estão incorporadas em outras estruturas de menu:

  • Para ver Pessoas individualmente, basta clicar no avatar da pessoa em uma conversa.

  • A funcionalidade Grupos agora é acessada no menu do usuário.

  • Coleções agora são gerenciadas em repositórios individuais.

  • Itens de Conteúdo são gerenciados em repositórios individuais. Tipos de conteúdo podem ser criados por administradores clicando em Ativos no menu Administração.

Release 18.1.5 (Contas Tradicionais do Cloud) — Março de 2018

Funcionalidade Atribuição Descrição
Sites: Modelos de conteúdo Usuários e administradores de sites

Agora é possível usar gabaritos de conteúdo para compartilhar modelos de conteúdo, o que inclui layouts de conteúdo, itens de conteúdo e ativos digitais necessários para o suporte a um modelo de conteúdo (como amostra de conteúdo).

O uso de um modelo de conteúdo permite que os usuários corporativos comecem a criar conteúdo editorial usando tipos e layouts de conteúdo com base no modelo. Eles podem publicar esse conteúdo em sites ou em outros canais disponíveis.

Você cria um modelo de conteúdo com base em um site existente. Ele pode então ser compartilhado com outros usuários. Os gabaritos de conteúdo devem ser criados com base em sites com ativos digitais e itens de conteúdo publicados.

Observe que os tipos de conteúdo não são criados quando um gabarito de site é importado de um pacote com conteúdo. Eles são criados quando um site é criado com base no gabarito de conteúdo. Portanto, os tipos de conteúdo não são propriedade do usuário que importa o gabarito do site. Em vez disso, os tipos de conteúdo são propriedade do usuário que cria um primeiro site com base nesse gabarito. Esse usuário pode então compartilhar o gabarito com outros usuários conforme necessário.
Sites: Desempenho aprimorado de publicação em sites Usuários dos sites

A ativação do site pode ser um processo demorado, mas uma nova funcionalidade entrega os arquivos do site diretamente de uma pasta de site dinâmica.
Sites: Mudança para o componente de lista de conteúdo Usuários dos sites

O componente Lista de Conteúdo foi aperfeiçoado com um campo de consulta adicional, permitindo que você use informações de filtro avançadas. Por exemplo, os operadores de pesquisa como eq (igual a) e le (menor ou igual a) podem ser usados para filtrar itens em uma lista de conteúdo.
Desempenho de upload do aplicativo de desktop Todos os usuários

Agora é mais rápido fazer o upload de conteúdo em redes estáveis. Agora você pode fazer upload de arquivos com mais de 5 GB, que era a limitação anterior.
