Fonctionnalité Rôle Description

Options de dimensionnement supplémentaires

Utilisateurs de site

Vous pouvez désormais utiliser d'autres dimensionnements pour de nombreux composants, tels que des galeries ou des images. Reportez-vous à Réglage de la taille des composants dans le guide Création d'expériences avec Oracle Content and Experience Cloud

Modifications apportées à la navigation dans l'éditeur de site

Utilisateurs de site

Lors de la modification d'un site, le menu de navigation à l'aide de la barre latérale a été modifié. Vous pouvez désormais accéder directement à vos ressources ainsi qu'à différents types de composant à partir du menu. Reportez-vous à Aperçu rapide de l'éditeur de site dans le guide Création d'expériences avec Oracle Content and Experience Cloud

Création de thèmes personnalisés

Développeurs de sites

Les développeurs peuvent créer des thèmes qui utilisent un sous-ensemble de composants spécifique. Vous pouvez accéder à ces "composants avec un thème" via le menu des composants dans le générateur de site. Reportez-vous à Présentation des thèmes dans le guide Création d'expériences avec Oracle Content and Experience Cloud et à Association de composants à des thèmes dans le guide Développement pour Oracle Content and Experience Cloud

Modèle du nouveau référentiel de ressources

Tous les utilisateurs

Les administrateurs de référentiel peuvent désormais créer plusieurs référentiels, chacun d'eux ayant ses propres stratégies de publication et de traduction. Reportez-vous à Présentation des référentiels de ressources dans le manuel Création d'expériences avec Oracle Content and Experience Cloud.

Les développeurs peuvent désormais utiliser le référentiel de ressources avec Sites Toolkit, intégration Developer Cloud Service. Reportez-vous à Utilisation du référentiel de ressources avec Sites Toolkit dans le manuel Développement pour Oracle Content and Experience Cloud.

Migration des référentiels de ressources en cours vers le nouveau modèle de référentiel

Administrateurs de référentiel

Les administrateurs de référentiel peuvent migrer les référentiels de ressources existants vers le nouveau modèle de référentiel en suivant les instructions détaillées de la section Migration à partir de ressources numériques traditionnelles vers de nouvelles ressources numériques dans le manuel Administration d'Oracle Content and Experience Cloud.

Nouveaux rôles utilisateur

Tous les utilisateurs

Deux nouveaux rôles utilisateur (Administrateur du référentiel et Développeur) ont été ajoutés et la documentation relative aux rôles a été améliorée. Reportez-vous à Rôles dans le manuel Administration d'Oracle Content and Experience Cloud.

Modifications apportées à la structure du menu

Tous les anciens utilisateurs

Les options de navigation pour le service ont été simplifiées. Ce que vous voyez dépend du rôle qui vous a été affecté et du type de votre service : standard ou d'entreprise.

  • Pour afficher les fonctionnalités de gestion de fichiers, cliquez sur Documents.

  • Pour accéder aux fonctionnalités liées aux conversations, cliquez sur Conversations.

  • Pour accéder aux fonctionnalités de création de sites, cliquez sur Sites.

  • Pour afficher les ressources numériques, cliquez sur Ressources.

  • Pour accéder aux modèles, aux thèmes et aux présentations, cliquez sur Développeur.

  • Pour accéder aux statistiques du service, cliquez sur Analyses.

Si un rôle d'administration vous a été affecté, les options supplémentaires incluent Paramètres, option utilisée pour définir les paramètres à l'échelle du service, et Ressources, option utilisée pour configurer les référentiels, les types de contenu, les canaux et les stratégies de localisation.

Personnes, groupes, collections, éléments de contenu

Tous les anciens utilisateurs

Ces fonctionnalités sont désormais intégrées à d'autres structures de menu :

  • Pour visualiser une personne en particulier, cliquez simplement sur son avatar dans une conversation.

  • La fonctionnalité Groupes est désormais accessible via le menu utilisateur.

  • Les collections sont désormais gérées dans des référentiels individuels.

  • Les éléments de contenu sont gérés dans des référentiels individuels. Les types de contenu peuvent être créés par les administrateurs en cliquant sur Ressources dans le menu Administration.

Traductions multilingues

Tous les utilisateurs

Les administrateurs de contenu peuvent définir des stratégies de traduction afin de permettre d'ajouter plusieurs langues pour les éléments de contenu. Reportez-vous à Localisation d'éléments de contenu dans le manuel Création d'expériences avec Oracle Content and Experience Cloud.

Nouveau modèle de départ JET

Utilisateurs de site

Il existe maintenant un modèle Oracle JET (boîte à outils d'extension Oracle JavaScript) pouvant être utilisé pour créer un site. JET est une boîte à outils open source modulaire qui utilise un ensemble de bibliothèques JavaScript open source. Pour plus d'informations, reportez-vous à Oracle JET.

Améliorations des sites

Utilisateurs de site
Plusieurs améliorations ont été apportées pour les utilisateurs qui créent des sites :

  • Les ressources numériques peuvent être sélectionnées et utilisées dans des composants personnalisés.

  • Les développeurs peuvent personnaliser la barre d'outils de l'éditeur utilisée dans les composants de sites.

  • Google Maps peut désormais être sélectionné comme option dans le composant de carte.

  • Le composant de recherche de contenu a été amélioré pour générer des résultats de recherche plus rapidement.

  • Oracle JET peut désormais être utilisé dans les mises en page et les pages.

Améliorations des sites pour une utilisation mobile

Utilisateurs de site
Des améliorations ont été apportées en vue de la création de sites destinés à un usage mobile, à savoir :

  • Lors de la prévisualisation d'un site pour une configuration mobile, les présentations et les écrans s'adaptent à la taille de l'appareil mobile choisi.

  • Les pages, les éléments de contenu ou les listes de contenu peuvent utiliser d'autres présentations lorsqu'ils sont conçus pour être utilisés sur un appareil mobile.

  • Des liens de carte peuvent être créés. Sur un appareil mobile, ils permettent d'ouvrir le lien dans les cartes natives.

Règles de réacheminement de site

Développeurs

Les développeurs peuvent désormais définir des règles pour rediriger les demandes utilisateur des anciennes URL vers les URL actuelles. Indiquer des réacheminements HTTP 30x pour les URL permet de conserver les signets ou les liens publiés d'une conception de site à une autre. Reportez-vous à Indication de réacheminements de site dans le manuel Développement pour Oracle Content and Experience Cloud.

Nouveau webhook de publication de contenu

Administrateurs

Utilisez le nouveau webhook pour publier des événements de notification de publication de contenu à partir d'Oracle Content and Experience Cloud vers un système externe. Reportez-vous à Configuration de webhooks dans le manuel Administration d'Oracle Content and Experience Cloud.

Filtres de groupe pour les données d'analyse

Administrateurs

Vous pouvez désormais filtrer les données d'analyse par groupe. Reportez-vous à Affichage des statistiques générales et à Affichage des modèles d'utilisation dans le manuel Administration d'Oracle Content and Experience Cloud.

Amélioration de la documentation

Tous les utilisateurs

La documentation a été modifiée pour mieux répondre aux besoins des publics spécifiques.

Le manuel Gestion du contenu à l'aide d'Oracle Content and Experience Cloud détaille comment utiliser l'aspect Documents du service, y compris comment partager et gérer des fichiers et des dossiers, et utiliser des groupes.

Le manuel Création d'expériences avec Oracle Content and Experience Cloud détaille comment créer des référentiels, gérer des stratégies de canal et de traduction, gérer des ressources numériques et créer des sites Web.

Modifications et ajouts liés aux API REST

Développeurs

L'API REST pour la gestion du contenu a été renommée API REST pour Documents. Reportez-vous à Présentation de l'API REST pour Documents dans le manuel Développement pour Oracle Content and Experience Cloud.

L'ancienne API REST pour la collaboration a été divisée en deux nouvelles API REST :

  • API REST pour les conversations

    Reportez-vous à Présentation de l'API REST pour les conversations dans le manuel Développement pour Oracle Content and Experience Cloud.

  • API REST pour les utilisateurs et les groupes

    Reportez-vous à Présentation de l'API REST pour les utilisateurs et les groupes dans le manuel Développement pour Oracle Content and Experience Cloud.

L'API REST pour Content Delivery a été ajoutée. Reportez-vous à Présentation de l'API REST pour Content Delivery dans le manuel Développement pour Oracle Content and Experience Cloud.

Kit SDK de diffusion de contenu

Développeurs

Le kit SDK de diffusion de contenu pour Oracle Content and Experience Cloud est un wrapper JavaScript léger qui interagit avec l'API REST de diffusion de contenu. Ce kit SDK en lecture seule récupère du contenu structuré, des ressources numériques et des présentations de contenu. Reportez-vous à Kit SDK de diffusion de contenu dans le manuel Développement pour Oracle Content and Experience Cloud.

Fonctionnalité Rôle Description
Sites : modèles de contenu Administrateurs et utilisateurs des sites

Vous pouvez désormais utiliser des modèles de contenu pour partager des modèles de contenu, comprenant des présentations de contenu, des éléments de contenu et des ressources numériques nécessaires pour la prise en charge d'un modèle de contenu (tel qu'un exemple de contenu).

L'utilisation d'un modèle de contenu permet aux utilisateurs entreprise de commencer à créer du contenu rédactionnel à l'aide des types de contenu et de leurs présentations à partir du modèle. Ils peuvent publier ce contenu sur des sites ou d'autres canaux disponibles.

Vous pouvez créer un modèle de contenu à partir d'un site existant. Vous pouvez ensuite partager ce contenu avec d'autres utilisateurs. Les modèles de contenu doivent être créés à partir de sites avec des ressources numériques et des éléments de contenu publiés.

Les types de contenu ne sont pas créés lorsqu'un modèle de site est importé à partir d'un package avec du contenu. Ils sont créés lorsqu'un site est créé à partir du modèle de contenu. Les types de contenu n'appartiennent donc pas à l'utilisateur qui importe le modèle de site, mais à l'utilisateur qui crée un premier site à partir de ce modèle. Cet utilisateur peut ensuite partager le modèle avec d'autres utilisateurs si nécessaire.
Sites : amélioration des performances de publication des sites Utilisateurs de site

L'activation de site peut être un processus chronophage. Toutefois, une nouvelle fonctionnalité permet de distribuer des fichiers de site directement à partir d'un dossier de sites dynamique.
Sites : modification du composant de liste de contenu Utilisateurs de site

Le composant de liste de contenu a été amélioré grâce à l'ajout d'un champ de requête, ce qui permet d'utiliser des informations de filtre améliorées. Par exemple, des opérateurs de recherche tels que eq (égal à) et le (inférieur ou égal à) peuvent être utilisés pour filtrer les éléments d'une liste de contenu.
Performances de téléchargement de l'application de bureau Tous les utilisateurs

Le téléchargement de contenu sur des réseaux stables est désormais plus rapide. Vous pouvez désormais télécharger des fichiers supérieurs à 5 Go (limitation antérieure).
Modification de prise en charge par l'application de bureau Tous les utilisateurs

Mac OS X 10.10 ("Yosemite") et les versions antérieures du système d'exploitation ne sont plus prises en charge.

  • A partir de Content and Experience Cloud, vous pouvez créer un modèle, exporter le modèle, puis importer le modèle dans Developer Cloud Service pour modifier le thème.
