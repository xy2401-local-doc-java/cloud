Alterações à estrutura de menus

Todos os utilizadores anteriores

As opções de navegação para o serviço foram simplificadas. O que vê depende do perfil de grupo que lhe foi atribuído e se o seu serviço é um serviço standard ou um serviço empresarial.

  • As funcionalidades de gestão de ficheiros são mostradas clicando em Documentos.

  • As funcionalidades relacionadas com conversações são acedidas clicando em Conversações.

  • Todas as funcionalidades de criação de sites são acedidas clicando em Sites.

  • Os ativos digitais são mostrados clicando em Ativos.

  • Os modelos, temas e disposições são acedidos clicando em Programador.

  • As estatísticas relacionadas com serviços são acedidas clicando em Sistema de Análise.

Se lhe tiver sido atribuído um perfil de grupo administrativo, as opções adicionais incluem Definições, utilizadas para especificar definições do serviço, e Ativos, utilizados para configurar repositórios, tipos de conteúdo, canais e perfis de localização.

Pessoas, Grupos, Coleções, Itens de Conteúdo

Todos os utilizadores anteriores

Estas funcionalidades são agora incorporadas noutras estruturas de menu:

  • Para ver Pessoas individualmente, basta clicar no avatar da pessoa numa conversação.

  • A funcionalidade Grupos é agora acedida através do seu menu de utilizador.

  • As Coleções são agora geridas nos repositórios individuais.

  • Os Itens de conteúdo são agora geridos nos repositórios individuais. Os Tipos de conteúdo podem ser criados por administradores clicando em Ativos no menu Administração.

Novo modelo inicial JET

Utilizadores de sites

Existe agora um modelo Oracle JET (JavaScript Extension Toolkit) que pode ser utilizado para criar um site. JET é um kit de ferramentas por módulos e de código fonte aberto utilizando uma coleção de bibliotecas JavaScript de códigos fonte abertos. Consulte Oracle Jet para obter detalhes.

Melhoramentos de sites

Utilizadores de sites
Foram realizados diversos melhoramentos para os utilizadores que estão a criar sites:

  • Os ativos digitais podem ser selecionados e utilizados em componentes customizados.

  • Os programadores podem customizar a barra de ferramentas do editor que é utilizada no componente Sites.

  • Os mapas Google podem agora ser selecionados como uma opção no componente Mapa.

  • O componente de pesquisa de conteúdo foi melhorado para produzir resultados de pesquisa mais rápidos.

  • O Oracle JET pode agora ser utilizado em disposições de páginas e em páginas.

Melhorias de sites para utilização em dispositivos móveis

Utilizadores de sites
Foram feitas melhorias para criar sites para uma definição de dispositivo móvel, incluindo:

  • Ao pré-visualizar um site para uma definição de dispositivo móvel, as disposições e os ecrãs são recetivos ao tamanho do dispositivo móvel escolhido.

  • As páginas, os itens de conteúdo ou as listas de conteúdo podem utilizar disposições alternativas quando concebidas para utilizar num dispositivo móvel.

  • As ligações do mapa podem ser criadas para abrir a ligação nos mapas nativos num dispositivo móvel.

Consulte Especificar Redirecionamentos de Site em Programar para o Oracle Content and Experience Cloud.

Sites: Modelos de conteúdo Utilizadores e admins de sites

Os modelos de conteúdo podem agora ser utilizados para partilhar modelos de conteúdo, que incluem disposições de conteúdo, itens de conteúdo e ativos digitais necessários para suportar um modelo de conteúdo (tal como conteúdo de exemplo).

Utilizar um modelo de conteúdo permite aos utilizadores empresariais começar a criar conteúdo editorial utilizando tipos de conteúdo e disposições do modelo. Podem publicar este conteúdo em sites ou noutros canais disponíveis.

Pode criar um modelo de conteúdo a partir de um site existente. Em seguida, pode ser partilhado com outros utilizadores. Os modelos de conteúdo devem ser criados a partir de sites com itens de conteúdo e ativos digitais publicados.

Tenha em atenção que não são criados tipos de conteúdo quando um modelo de site é importado a partir de um pacote com conteúdo. São criados quando um site é criado a partir do modelo de conteúdo. Por conseguinte, os tipos de conteúdo não pertencem ao utilizador que importa o modelo do site. Em vez disso, os tipos de conteúdo são propriedade do utilizador que cria um primeiro site a partir desse modelo. Esse utilizador poderá, então, partilhar o modelo com outros utilizadores, conforme necessário.
Sites: Desempenho de publicação de sites melhorado Utilizadores de sites

A ativação de sites pode ser um processo moroso, mas uma nova funcionalidade fornece ficheiros do site diretamente a partir de uma pasta do site dinâmica.
Sites: Mudar para componente Lista de Conteúdos Utilizadores de sites

O componente Lista de Conteúdos tem vindo a ser melhorado com um campo adicional de consulta, que lhe permite utilizar informações de filtro avançadas. Por exemplo, os operadores de pesquisa como eq (igual a) e le (menor que ou igual a) podem ser utilizados para filtrar itens numa lista de conteúdos.
Desempenho do carregamento da aplicação do ambiente de trabalho Todos os utilizadores

O carregamento de conteúdo em redes estáveis é agora mais rápida. Pode agora carregar ficheiros superiores à limitação anterior de 5 GB.

  • Ao começar no Content and Experience Cloud, pode criar um modelo, exportar o modelo e, em seguida, importar o modelo para o Developer Cloud Service para editar o tema.
