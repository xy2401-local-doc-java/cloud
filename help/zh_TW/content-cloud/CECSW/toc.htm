Oracle Content and Experience Cloud 新功能

18.3.5 (通用點數模型) — 2018 年 9 月

功能 角色 描述

其他調整大小選項

網站使用者

您現在可以更改許多元件的大小，例如圖庫或影像。您可以繼續使用像素來指定大小，或使用其他單位，例如階層式樣式表 (CSS) 中的單位。請參閱運用 Oracle Content and Experience Cloud 建立體驗中的調整元件大小

網站編輯器導覽變更

網站使用者

編輯網站時的提要欄位導覽功能表已有所改變。現在可以直接從功能表存取您的物件和其他類型元件。請參閱運用 Oracle Content and Experience Cloud 建立體驗中的網站編輯器快速總覽

可以建立自訂主題

網站開發人員

開發人員可以建立使用特定元件子集的主題。您可以透過網站產生器中的元件功能表存取這些主題元件。請參閱運用 Oracle Content and Experience Cloud 建立體驗中的瞭解主題和 Developing for Oracle Content and Experience Cloud 中的建立元件和主題的關聯

版本 18.3.3 (通用點數模型) — 2018 年 8 月

功能 角色 描述

新的物件儲存區域模型

所有使用者

儲存區域管理員現在可以建立多個儲存區域，每個區域可以有各自的發布和翻譯規定。請參閱運用 Oracle Content and Experience Cloud 建立體驗中的瞭解物件儲存區域。

開發人員現在可以透過 Developer Cloud Service 整合的網站工具程式來使用物件儲存區域。請參閱 Developing for Oracle Content and Experience Cloud 中的 Using the Asset Repository with Sites Toolkit。

將目前的物件儲存區域移轉至新的儲存區域模型

儲存區域管理員

儲存區域管理員可以依照Oracle Content and Experience Cloud 管理方法的從傳統數位物件移轉成新型數位物件中詳述的指示，將現有的物件儲存區域移轉到新的儲存區域模型。

新使用者角色

所有使用者

已新增兩種新的使用者角色 – 儲存區域管理員和開發人員，並且改進了角色相關文件。請參閱 Oracle Content and Experience Cloud 管理方法中的角色。

功能表結構變更

所有舊版使用者

服務的導覽選項經過簡化。使用者可以見到的內容取決於所指定的角色，以及服務是標準服務還是企業服務。

  • 按一下文件即會顯示檔案管理功能。

  • 按一下對話即可取用對話相關功能。

  • 按一下網站即可取用所有網站建立功能。

  • 按一下物件即會顯示數位物件。

  • 按一下開發人員即可取用樣板、主題和版面配置。

  • 按一下分析即可取用服務相關統計資料。

使用者若具備管理角色，則另有其他選項，包括可用來設定整體服務設定值的設定值，以及可用來設定儲存區域、內容類型、通道以及本地化規定的物件。

人員、群組、集合、內容項目

所有舊版使用者

這些功能現在都併入其他功能表結構中：

  • 若要檢視個別人員，只要按一下對話中該人員的大頭貼即可。

  • 群組功能現在透過使用者功能表取用。

  • 集合現在於個別的儲存區域中管理。

  • 內容項目也在個別儲存區域中管理。管理員可以按一下「管理」功能表中的「物件」來建立內容類型。

多語言翻譯

所有使用者

內容管理員可以設定翻譯規定，以便用於對內容項目新增多種語言。請參閱運用 Oracle Content and Experience Cloud 建立體驗中的本地化內容項目。

新的 JET 入門樣板

網站使用者

現在可以使用 Oracle JET (JavaScript Extension Toolkit) 樣板來建立網站。JET 是使用開源 JavaScript 程式庫集合的模組化開源工具程式。請參閱 Oracle Jet 瞭解詳細資訊。

網站增強功能

網站使用者
以下是為建立網站的使用者提供的增強功能：

  • 可以在自訂元件內選取並使用數位物件。

  • 開發人員可以自訂網站元件中使用的編輯器工具列。

  • 地圖元件中現在提供 Google 地圖選項。

  • 內容搜尋元件現在可以更快的速度產生搜尋結果。

  • 現在可以在頁面版面配置和頁面中使用 Oracle JET。

供行動使用的網站增強功能

網站使用者
針對建立包含行動裝置設定的網站推出部分增強功能，其中包括：

  • 以行動裝置設定預覽網站時，版面配置和畫面都會根據所選擇行動裝置的大小顯示。

  • 針對行動裝置使用設計時，可以使用頁面、內容項目或內容清單的替代版面配置。

  • 建立地圖連結時，可以選擇在行動裝置的原生地圖中開啟該連結。

網站重導規則

開發人員

開發人員現在可以指定規則，將使用者要求從舊的 URL 重導至目前的 URL。為 URL 指定 HTTP 30x 重導可以在網站重新設計時繼續使用原來的書籤和已發布的連結。請參閱 Developing for Oracle Content and Experience Cloud 中的 Specifying Site Redirects。

新的內容發布 Webhook

管理使用者

使用新的 Webhook，從 Oracle Content and Experience Cloud 將內容發布通知事件張貼至外部系統。請參閱 Oracle Content and Experience Cloud 管理方法中的設定 Webhook 。

對分析資料進行群組篩選

管理使用者

您現在可以依群組篩選分析資料。請參閱 Oracle Content and Experience Cloud 管理方法 中的檢視一般統計資料和檢視使用狀況模式。

文件增強功能

所有使用者

文件功能現在更能夠滿足特定對象的需求。

運用 Oracle Content and Experience Cloud 管理內容詳細說明如何使用服務的「文件」層面，包括如何共用檔案和資料夾、使用群組，以及管理檔案和資料夾。

運用 Oracle Content and Experience Cloud 建立體驗詳細說明如何建立儲存區域、管理通道和翻譯規定，以及如何管理數位物件和如何建立網站。

REST API 變更和新增項目

開發人員

REST API for Content Management 現已重新命名為 REST API for Documents。請參閱 Developing for Oracle Content and Experience Cloud 中的 Overview of the REST API for Documents。

原本的 REST API for Collaboration 已分成兩個新的 REST API：

  • REST API for Conversations

    請參閱 Developing for Oracle Content and Experience Cloud中的Overview of the REST API for Conversations。

  • REST API for Users and Groups

    請參閱 Developing for Oracle Content and Experience Cloud 中的 Overview of the REST API for Users and Groups

新增了 REST API for Content Delivery。請參閱 Developing for Oracle Content and Experience Cloud 中的 Overview of the REST API for Content Delivery。

Content Delivery SDK

開發人員

適用於 Oracle Content and Experience Cloud 的 Content Delivery SDK 是一種能夠和 Content Delivery REST API 互動的輕量型 JavaScript 包裝函式。此唯讀 SDK 可擷取結構化內容、數位物件及內容版面配置。請參閱 Developing for Oracle Content and Experience Cloud 中的 Content Delivery SDK。

版本 18.1.5 (傳統雲端帳戶) — 2018 年 3 月

功能 角色 描述
網站：內容樣板 網站使用者和管理員

內容樣板現在可用於共用內容模型，其中包括支援內容模型 (例如範例內容) 所需的內容版面配置、內容項目以及數位物件。

使用內容樣板，可讓企業使用者開始使用樣板中的內容類型和版面配置來建立評論內容。他們可以將此內容發布到網站或其他可用的通道。

您可以從現有網站建立內容樣板。樣板建立後就可與其他使用者共用。內容樣板應該從含有已發布內容項目和數位物件的網站建立。

注意，若從含有內容的封裝項目匯入網站樣板，並不會建立內容類型。若從內容樣板建立網站，才會建立內容類型。因此，內容類型並不是由匯入網站樣板的使用者擁有，而是由從該樣板建立第一個網站的使用者所擁有。該使用者可視需要與其他使用者共用該樣板。
網站：增強網站發布效能 網站使用者

啟用網站是一項費時的處理作業，但有了這個新功能，就可以直接從動態網站資料夾傳遞網站檔案。
網站：內容清單元件變更 網站使用者

有了額外的查詢欄位，內容清單元件變得更加完善，讓您能夠使用強化的篩選資訊。例如，像是 eq (等於) 和 le (小於或等於) 這一類的搜尋運算子，可以用來篩選內容清單中的項目。
桌面應用程式上傳效能 所有使用者

如今在穩定網路上傳內容的速度更快。現在您可以上傳大於先前 5GB 限制的檔案。
桌面應用程式支援變更 所有使用者

已不再支援 Mac OS X 10.10 ("Yosemite") 和舊版的作業系統。
Developer Cloud Service 整合

開發人員

您現在可以為 Developer Cloud Service 中的 Content and Experience Cloud 開發樣板和主題。

  • 從 Developer Cloud Service 開始的話，您可以建立樣板、編輯主題，然後匯出樣板，最後匯入 Content and Experience Cloud 中。

  • 從 Content and Experience Cloud 開始的話，您可以建立樣板、匯出樣板，然後將樣板匯入 Developer Cloud Service 以編輯主題。

跨平台的命令行公用程式 (CEC) 提供整合式的協助，用於搭配範例的使用來執行命令。本機測試控制程式支援執行與測試所有類型的 Content and Experience Cloud 元件和樣板，包括內容樣板。

請參閱 Developing for Oracle Content and Experience Cloud 中的 Developing for Content and Experience Cloud with Developer Cloud Service 和 Developing Templates with Developer Cloud Service。
