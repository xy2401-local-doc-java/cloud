内容和体验云的新增功能

2018 年 9 月

Oracle Content and Experience Cloud 中的新增功能

18.3.5（通用储值模型）— 2018 年 9 月

功能 角色 说明

其他大小调整选项

站点用户

现在可以对许多组件（例如图库或图像）使用替代大小调整。可继续以像素为单位指定大小，也可以使用其他单位，例如级联样式表 (Cascading Style Sheet, CSS) 中的那些单位。请参见《使用 Oracle Content and Experience Cloud 创建体验》中的“调整组件的大小”

站点编辑器导航更改

站点用户

编辑站点时子工具栏导航菜单已更改。现在可以直接从菜单访问您的资产，也可以从菜单访问不同类型的组件。请参见《使用 Oracle Content and Experience Cloud 创建体验》中的“站点编辑器快速概览”

可以创建定制主题

站点开发人员

开发人员可以创建使用特定组件子集的主题。可以通过站点构建器中的组件菜单访问这些“主题化组件”。请参见《使用 Oracle Content and Experience Cloud 创建体验》中的“了解主题”以及《Developing for Oracle Content and Experience Cloud》中的 "Associating Components with Themes"

发行版 18.3.3（通用储值模型）— 2018 年 8 月

功能 角色 说明

新资产资料档案库模型

所有用户

资料档案库管理员现在可以创建多个资料档案库，每个资料档案库具有自己的发布和翻译策略。请参见《使用 Oracle Content and Experience Cloud 创建体验》中的“了解资产资料档案库”。

现在，开发人员可以将资产资料档案库与站点工具箱结合使用，即开发人员云服务集成。请参见《Developing for Oracle Content and Experience Cloud》中的“将资产资料档案库与站点工具箱结合使用”。

将当前资产资料档案库迁移到新的资料档案库模型

资料档案库管理员

资料档案库管理员可以按照《管理 Oracle Content and Experience Cloud》中“从传统数字资产迁移到新的数字资产”详述的说明操作，将现有资产资料档案库迁移到新的资料档案库模型。

新用户角色

所有用户

添加了两个新的用户角色（资料档案库管理员和开发人员），并且改进了角色文档。请参见《管理 Oracle Content and Experience Cloud》中的“角色”。

对菜单结构进行了更改

所有以前的用户

简化了服务的导航选项。您所看到的内容取决于向您分配的角色以及您的服务是标准服务还是企业服务。

  • 单击文档可显示文件管理功能。

  • 单击对话可访问与对话相关的功能。

  • 单击站点可访问所有站点构建功能。

  • 单击资产可显示数字资产。

  • 单击开发人员可访问模板、主题和布局。

  • 单击分析可访问与服务相关的统计信息。

如果已向您分配了管理角色，则其他选项包括设置（用于设置服务范围的设置）以及资产（用于设置资料档案库、内容类型、通道和本地化策略）。

人员、组、集合、内容项

所有以前的用户

这些功能现在并入到其他菜单结构中：

  • 要查看单独的人员，只需在对话中单击此人的头像。

  • 组功能现在可通过用户菜单访问。

  • 现在集合在单独资料档案库中管理。

  • 内容项在单独资料档案库中管理。管理员可通过在“管理”菜单上单击“资产”来创建内容类型。

多语言翻译

所有用户

内容管理员可以设置翻译策略，这些策略可用于为内容项添加多种语言。请参见《使用 Oracle Content and Experience Cloud 创建体验》中的“本地化内容项”。

新 JET 入门级模板

站点用户

现在提供了 Oracle JET（JavaScript 扩展工具包）模板用于创建站点。JET 是一种模块化的开源工具包，它使用开源 JavaScript 库的集合。有关详细信息，请参见 Oracle Jet。

站点增强功能

站点用户
为创建站点的用户提供了多种增强功能：

  • 可在定制组件中选择和使用数字资产。

  • 开发人员可定制在站点组件中使用的编辑器工具栏。

  • 现在可在“地图”组件中将 Google Maps 作为选项来选择。

  • 对内容搜索组件进行了增强，以更快生成搜索结果。

  • Oracle JET 现在可在页布局和页中使用。

用于移动用途的站点增强功能

站点用户
提供了增强功能以便为移动设置创建站点，包括：

  • 在预览移动设置的站点时，布局和屏幕根据所选移动设备的大小做出响应。

  • 当设计用于移动设备时，页、内容项或内容列表可使用替代布局。

  • 可以创建地图链接，它们可在移动设备上打开本机地图中的链接。

站点重定向规则

开发人员

开发人员现在可以指定规则，将用户请求从旧 URL 重定向到当前 URL。为 URL 指定 HTTP 30x 重定向可以在站点重新设计方案之间维护书签或已发布的链接。请参见《Developing for Oracle Content and Experience Cloud》中的 "Specifying Site Redirects"。

新的内容发布 Webhook

管理用户

使用新的 Webhook 将 Oracle Content and Experience Cloud 中的内容发布通知事件发布到外部系统。请参见《管理 Oracle Content and Experience Cloud》中的“配置 Webhook ”。

分析数据的组筛选器

管理用户

现在可以按组筛选分析数据。请参见《管理 Oracle Content and Experience Cloud》中的“查看一般统计信息”和“查看使用模式”。

文档增强功能

所有用户

修改了文档以更好地满足特定受众的需求。

《通过 Oracle Content and Experience Cloud 管理内容》详细说明如何使用服务的文档方面，包括如何共享文件和文件夹、使用组以及管理文件和文件夹。

《使用 Oracle Content and Experience Cloud 创建体验》详细说明如何创建资料档案库、管理通道和翻译策略、如何管理数字资产以及如何创建网站。

REST API 更改和添加

开发人员

适用于内容管理的 REST API 已重命名为适用于文档的 REST API。请参见《Developing for Oracle Content and Experience Cloud》中的 "Overview of the REST API for Documents"。

以前的适用于协作的 REST API 已分成两个新的 REST API：

  • 适用于对话的 REST API

    请参见《Developing for Oracle Content and Experience Cloud》中的"Overview of the REST API for Conversations"。

  • 适用于用户和组的 REST API

    请参见《Developing for Oracle Content and Experience Cloud》中的 "Overview of the REST API for Users and Groups"。

添加了 REST API for Content Delivery。请参见《Developing for Oracle Content and Experience Cloud》中的 "Overview of the REST API for Content Delivery"。

Content Delivery SDK

开发人员

适用于 Oracle Content and Experience Cloud 的内容传送 SDK 是一个轻型 JavaScript 包装程序，它与内容传送 REST API 交互。此只读 SDK 可检索结构化内容、数字资产和内容布局。请参见《Developing for Oracle Content and Experience Cloud》中的 "Content Delivery SDK"。

发行版 18.2.5（传统 Cloud 账户）— 2018 年 6 月

重要通知：即将对资产资料档案库进行的更改：在即将发行的面向通用储值模型客户的发行版中，将对资产资料档案库进行重大更改。新的资产模型需要将内容从当前模型迁移到新模型。根据资产资料档案库的大小以及使用资产的方式，这可能是涉及到多个步骤的详细流程。在可行的情况下，您可能需要将新资产项目的添加推迟到新资产模型发布之后。

发行版 18.1.5（传统 Cloud 账户）— 2018 年 3 月

功能 角色 说明
站点：内容模板 站点用户和管理员

内容模板现在可用于共享内容模型，这包括支持内容模型（例如示例内容）所需的内容布局、内容项和数字资产。

企业用户可以通过内容模板，使用模板的内容类型和布局开始创建编辑内容。他们可将此内容发布到站点或其他可用渠道。

您可从现有站点创建内容模板。然后将它与其他用户共享。应该从站点使用已发布的内容项和数字资产创建内容模板。

请注意，在从带有内容的程序包导入站点模板时并未创建内容类型。它们在从内容模板创建站点时创建。因此，内容类型不由导入站点模板的用户拥有。相反，内容类型由从该模板创建第一个站点的用户拥有。然后，用户可以根据需要与其他用户共享模板。
站点：增强站点发布性能 站点用户

站点激活过程可能会耗费较长的时间，不过有一个新功能可以直接将站点文件传送到动态站点文件夹。
站点：对内容列表组件的更改 站点用户

通过额外的查询字段来改进内容列表组件，让您可以使用增强的筛选信息。例如，可以使用 eq（等于）和 le（小于等于）等搜索运算符来筛选内容列表中的项。
桌面应用程序上载性能 所有用户

现在，通过稳定网络上载内容的速度更快。您现在可以上载超过 5GB 的文件，这是以前的限制值。
桌面应用程序支持更改 所有用户

不再支持 Mac OS X 10.10 ("Yosemite") 和操作系统较早的版本。
开发人员云服务集成

开发人员

现在可在开发人员云服务中为内容和体验云开发模板和主题。

  • 从开发人员云服务开始，您可以创建模板、编辑主题，然后导出模板以便导入到内容和体验云中。

  • 从内容和体验云开始，您可以创建模板、导出模板，然后将模板导入到开发人员云服务中来编辑主题。

跨平台命令行实用程序 CEC 为运行命令提供了集成帮助以及示例。本地测试工具支持运行和测试所有类型的内容和体验云组件及模板，包括内容模板。

请参见《Developing for Oracle Content and Experience Cloud》中的 "Developing for Content and Experience Cloud with Developer Cloud Service" 和 "Developing Templates with Developer Cloud Service"。

跨源资源共享 (CORS) 设置改进 开发人员，管理员 CORS 设置适用于所有 Oracle Content and Experience Cloud 调用（文档、社交和内容即服务）。请参见《管理 Oracle Content and Experience Cloud》中的“启用跨源资源共享 (CORS)”。

所有客户端：新共享流 用户 新共享流简化了共享的过程。现在通过在 Web 浏览器中直接选择菜单选项，您可以创建共享链接或者将成员添加到文件夹，与在移动客户端中的操作类似。公共链接的选项在默认情况下设置，使得创建该类型的链接非常简单。使用速成教学或参见《使用 Oracle Content and Experience Cloud》中的“如何共享文件？”和“如何共享文件夹？”。
所有客户端：组支持 用户，管理员 您可以创建人员组，或者使用服务管理员已经为贵组织创建的组。您可以将组作为成员添加到对话或文件夹，或者将内容类型与组共享。通过这种方式，您可以一次性与许多人分享信息。使用速成教学或参见《使用 Oracle Content and Experience Cloud》中的“如何使用组？”。管理员的最佳实践在《管理 Oracle Content and Experience Cloud》的为组织创建组标准部分中介绍。
站点：新共同浏览组件 站点管理员，用户 共同浏览功能是与 Oracle Cobrowse Cloud Service 的集成，用于在站点屏幕上添加按钮，从而通过可信连接启用屏幕共享。请参见《使用 Oracle Content and Experience Cloud》中的“如何在页上使用 Cobrowse？”。
站点：段落组件中的图像 站点用户 现在可以在段落组件中添加图像。请参见《使用 Oracle Content and Experience Cloud》中的“如何使用段落？”。
站点：新选项卡式部分布局 站点用户 使用新的选项卡式布局可以在一个布局中创建内容的选项卡式“页”。请参见《使用 Oracle Content and Experience Cloud》中的“如何使用选项卡式部分布局？”。
站点和内容项：新预览选项 站点用户 创建内容项和使用编辑器时有新选项可用。您可以创建定制大小，并使用标尺来轻松更改预览大小。请参见《使用 Oracle Content and Experience Cloud》中的“站点编辑器速成教学”和“如何创建内容项？”。
站点：改进高速缓存性能 站点用户，站点开发人员 新功能可以控制浏览器中的高速缓存行为，从而在构建站点以及使用已发布站点时动态更新站点资源。请参见《Developing for Oracle Content and Experience Cloud》中的Leveraging Cache to Improve Performance。
