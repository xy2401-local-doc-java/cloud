Änderungen an der Menüstruktur

Alle bisherigen Benutzer

Die Navigationsoptionen für den Service wurden vereinfacht. Die angezeigten Optionen sind neben der zugewiesenen Rolle davon abhängig, ob es sich um einen Standardservice oder einen Enterprise-Service handelt.

  • Die Dateiverwaltungsfeatures werden durch Klicken auf Dokumente angezeigt.

  • Die Unterhaltungsfeatures werden durch Klicken auf Unterhaltungen angezeigt.

  • Alle Features zur Siteerstellung werden durch Klicken auf Sites angezeigt.

  • Digitale Assets werden durch Klicken auf Assets angezeigt.

  • Vorlagen, Themes und Layouts werden durch Klicken auf Entwickler angezeigt.

  • Servicestatistiken werden durch Klicken auf Analysen angezeigt.

Wenn Ihnen eine administrative Rolle zugewiesen wurde, sind außerdem die Optionen Einstellungen zum Festlegen serviceweiter Einstellungen und Assets zum Einrichten von Repositorys, Inhaltstypen, Kanälen und Lokalisierungs-Policys verfügbar.

Personen, Gruppen, Sammlungen, Inhaltselemente

Alle bisherigen Benutzer

Diese Features befinden sich jetzt innerhalb anderer Menüstrukturen:

  • Klicken Sie zum Anzeigen einzelner Personen in einer Unterhaltung auf deren Avatare.

  • Das Feature Gruppen ist jetzt über das Benutzermenü zugänglich.

  • Sammlungen werden jetzt in den jeweiligen Repositorys verwaltet.

  • Inhaltselemente werden in den jeweiligen Repositorys verwaltet. Inhaltstypen können von Administratoren durch Klicken auf "Assets" im Menü "Administration" erstellt werden.

Release 18.1.5 (Herkömmliche Cloud-Accounts) - März 2018

Feature Rolle Beschreibung
Sites: Inhaltsvorlagen Sitebenutzer und Admins

Inhaltsvorlagen können jetzt zum Freigeben von Inhaltsmodellen verwendet werden. Diese Modelle umfassen Inhaltslayouts, Inhaltselemente und digitale Assets, die zur Unterstützung eines Inhaltsmodells benötigt werden (beispielsweise Beispielinhalt).

Mithilfe einer Inhaltsvorlage können Enterprise-Benutzer anhand von Inhaltstypen und Layouts aus der Vorlage mit dem Erstellen redaktionellen Inhalts beginnen. Sie können diesen Inhalt auf Sites oder anderen verfügbaren Kanälen veröffentlichen.

Eine Inhaltsvorlage wird aus einer vorhandenen Site erstellt. Anschließend kann sie für andere Benutzer freigegeben werden. Inhaltsvorlagen sollten aus Sites mit veröffentlichten Inhaltselementen und digitalen Assets erstellt werden.

Beachten Sie, dass beim Import einer Sitevorlage aus einem Paket mit Inhalt keine Inhaltstypen erstellt werden. Sie werden erstellt, wenn eine Site aus der Inhaltsvorlage erstellt wird. Daher ist der Benutzer, der die Sitevorlage importiert, nicht der Eigentümer der Inhaltstypen. Stattdessen gehören die Inhaltstypen dem Benutzer, der eine erste Site aus der entsprechenden Vorlage erstellt. Dieser Benutzer kann die Vorlage dann nach Bedarf für andere Benutzer freigeben.
Sites: Verbesserte Performance beim Veröffentlichen von Sites Sitebenutzer

Die Aktivierung einer Site kann viel Zeit in Anspruch nehmen. Dank eines neuen Features werden Sitedateien jedoch direkt aus einem dynamischen Siteordner bereitgestellt.
Sites: Änderung an der Inhaltslistenkomponente Sitebenutzer

Die Inhaltslistenkomponente wurde mit einem zusätzlichen Abfragefeld verbessert, über das Sie noch detailliertere Filterinformationen angeben können. Beispiel: Mithilfe von Suchoperatoren wie "EQ" (gleich) und "LE" (kleiner gleich) können Elemente in einer Inhaltsliste gefiltert werden.
Uploadperformance der Desktop-App Alle Benutzer
