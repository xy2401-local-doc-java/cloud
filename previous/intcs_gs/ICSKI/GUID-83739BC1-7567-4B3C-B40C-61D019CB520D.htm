Basic Authentication Fields are Unavailable when Updating a SOAP Adapter Connection after Importing an Integration

After importing an IAR file that was exported from a 16.2.5 release, while updating a SOAP Adapter Connection that uses Basic Authentication, the Username and Password fields are not available in the Configure Security dialog. For a trigger connection, these credentials are not required because this is a SOAP endpoint. The credentials to access the endpoint are always the Integration Cloud Service runtime user and password. In previous releases, the inbound SOAP Adapter connection accepted username and password changes.
