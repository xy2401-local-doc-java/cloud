What You Can Do from the Adobe eSign Adapter Operations Page

The table provides definitions for the Adobe eSign API operations that can be performed on the invoke connection. These operations are listed on the Adobe eSign Adapter Operations page.

Operation Description Introduced in Release
Send Agreement for Signature Creates an agreement and then sends it for a signature. 16.1.5
Get Agreement List for User Returns a list of agreements for a specific user. 16.1.5
Get Agreement Status Returns the latest status of a specific agreement. 16.1.5
Get Document Ids of Agreement Returns the IDs of the primary and supporting documents for a specific agreement. 16.1.5
Get Document URL Returns the URL of a specific document. 16.1.5
Cancel an Agreement Cancels an agreement and changes its status to cancel. 16.1.5
Delete an Agreement Deletes all documents associated with an agreement. 16.1.5
Get document of an agreement Returns the file stream of a document of an agreement. 16.1.5
Get information of the documents associated with an agreement Returns a single, combined PDF document for the documents associated with an agreement. 16.1.5
Get agreement form data Returns the data entered by the user into interactive form fields when they signed the agreement. 16.1.5
Get the audit trail of an agreement Returns the audit trail of an agreement identified by the agreement ID. 16.1.5
Upload a document Uploads a document and obtains returns the ID of the document. 16.1.5
Get User Workflows Returns workflows for a user. 16.1.5
Get details of a Workflow Returns the details of a workflow. 16.1.5
Create and Send an agreement out for signature Creates an agreement, sends it for signatures, and returns the agreement ID in the response to the client. 16.1.5
Send Agreement for signature to multiple recipients Sends an agreement to multiple recipients for their signature. Each recipient is sent a copy of the agreement for review and authorization. 16.1.5
