Adding an Exposed Service and Using the Adapter Wizard

In JDeveloper, you add an inbound adapter to an application as an exposed service.

  1. Drag and drop the SampleCloudAdapter from the component palette (Cloud Category) to the Exposed Services section.

    Description of the illustration GUID-DCD44826-0E0F-4535-9587-FCE7AF798274-default.png
  2. The wizard pops up and asks you to configure inbound-related information. The wizard starts with the Welcome page, which asks you to type a SampleCloudAdapter reference name.

    Description of the illustration GUID-29DD2838-0006-45BE-9F65-6725242B84D7-default.png
  3. Click Next. The Connection page appears and asks you to provide the source WSDL reference.
    You can also enter security credentials using an Authentication Key.

    Description of the illustration GUID-189B007E-D91F-4562-9C5C-E4C8F1E3246B-default.png
  4. To test connectivity to the service, click Test.

    A success or failure message is displayed in the Text area.


    Description of the illustration GUID-11F20EDF-2ED2-4DBC-9BAA-F7010BB7343B-default.png
  5. Click Next. Choose the business object you would like to receive from the Oracle Sample Cloud Service as the request for this integration flow.

    Description of the illustration GUID-BE3C3CE1-B55F-4374-AFDD-606C32034532-default.png
  6. Choose the business object you would like to receive from Oracle Sample Cloud Service as the response from this integration flow.
  7. Choose Immediate for the synchronous integration pattern.
    The Immediate response mode specifies a conversational, synchronous response, while the Delayed mode specifies an asynchronous response.

    Description of the illustration GUID-19D719FC-CBC6-4A80-9E37-B867EEEA61E2-default.png
  8. Click Next. The Summary page appears and displays the items selected during configuration.

    Description of the illustration GUID-C8C6C055-C832-4994-A766-21762AC42E87-default.png
  9. Click Finish to generate the runtime artifacts and references.
    Note: If clicking Finish results in a NullPointerException in the UI thread, it means that you didn’t implement the updateBackEndModel() method for one or more of your wizard pages.

    Description of the illustration GUID-446B40BE-2962-4B82-9EAF-B824A8F47C6B-default.png

These files are generated:

  • SampleService_REQUEST.jca (Cloud Application Configuration File)

  • SampleService_REQUEST.WSDL (Integration WSDL)

  • One of the following:

    • SampleService_ENDPOINT.WSDL (SOAP WSDL – exposed as service)

    • SampleService_ENDPOINT.WADL (REST WADL – exposed as service)

See Creating a BPEL Process to Consume the Adapter.
