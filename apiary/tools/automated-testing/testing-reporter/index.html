Apiary test reporter

Navigating through and investigating complex HTTP transactions is laborious—the Apiary reporter is striving to make it easier.

It’s divided to three tabs-Local Development, Continuous Integration, Post-Deployment-depending on where the tests were run. Each tab consists of three columns - list of all saved runs, test steps of the selected run and details of the selected test step which include not only the expected and real response but it’s diff as well so you can see clearly why the test is failing.
